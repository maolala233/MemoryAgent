Core Data Structures
====================

.. note::

   This document has been migrated to :doc:`/shared/data-structures-reference`. This page will be removed in a future version, please update your bookmarks.

This section introduces the core data structures of the Mandol memory system, including memory units, memory spaces, semantic index (SemanticMap), and semantic relation graph (SemanticGraph).

.. _en-data-structure-memory-unit:

MemoryUnit
----------

``MemoryUnit`` is the smallest storage unit in the memory system. Each unit contains raw data, metadata, and optional vector representations.

.. note::

   **Insertion Mode**: The system automatically embeds the following fields in ``MemoryUnit.raw_data``:

   - ``text_content``: Text content, automatically converted to dense vectors
   - ``image_path``: Image path, automatically converted to image vectors

   Other arbitrary fields (e.g., ``speaker``, ``source``, ``session_id``) are stored as metadata but NOT automatically embedded.

Core Attributes
^^^^^^^^^^^^^^^

.. code-block:: python

   from mandol import MemoryUnit, Uid

   unit = MemoryUnit(
       uid=Uid("dialogue_001"),
       raw_data={
           "text_content": "Zhang San went to Beijing on a business trip",
           "speaker": "user",
       },
       metadata={
           "timestamp": "2024-01-15T10:00:00",
           "spaces": ["root_base_memory"],
           "session_id": "session_001",
       },
       embedding=None,
       sparse_embedding=None,
   )

- ``uid``: Unique identifier, type ``Uid`` (defined in ``mandol/domain/types.py``)
- ``raw_data``: Raw data dictionary; system auto-embeds ``text_content`` and ``image_path`` fields
- ``metadata``: System metadata, includes timestamps, space information, etc.
- ``embedding``: Dense vector representation (generated by EmbeddingProvider), type ``Optional[numpy.ndarray]``
- ``sparse_embedding``: Sparse vector representation (generated by SparseIndex), type ``Optional[numpy.ndarray]``

Key Methods
^^^^^^^^^^^

- ``to_dict()``: Serialize to dictionary for persistence
- ``from_dict(data)`` (classmethod): Deserialize from dictionary to MemoryUnit
- ``get_user_metadata()``: Get user-defined metadata (excluding ``_system_`` prefixed fields)
- ``touch()``: Update ``_system_updated_at`` timestamp

Usage Example
^^^^^^^^^^^^^

.. note::

   The following example demonstrates the creation of a **base dialogue unit**, added by the user via ``system.add(unit)``.
   Multi-perspective memory units (entities, events, summaries, insights, etc.) are automatically constructed by the system in ``build_high_level()``
   and **should not** be manually created or inserted, otherwise UID conflicts or relationship inconsistencies may occur.

.. code-block:: python

   from mandol import MemorySystem, MemoryUnit, Uid

   system = MemorySystem()

   unit = MemoryUnit(
       uid=Uid("dialogue_001"),
       raw_data={"text_content": "Zhang San went to Beijing on a business trip", "speaker": "user"},
       metadata={"timestamp": "2024-01-15T10:00:00"},
   )
   system.add(unit)

   data = unit.to_dict()
   restored_unit = MemoryUnit.from_dict(data)

.. _en-data-structure-memory-space:

MemorySpace
-----------

``MemorySpace`` is a logical space for organizing MemoryUnits, supporting tree-like hierarchical relationships. Each space can contain multiple memory units and multiple subspaces.

Core Attributes
^^^^^^^^^^^^^^^

- ``name``: Space name, type ``SpaceName``
- ``unit_uids``: Set of contained memory unit UIDs, type ``Set[Uid]``
- ``child_spaces``: Set of subspace names, type ``Set[SpaceName]``
- ``summary_text``: Space summary text (optional)
- ``summary_embedding``: Summary vector (optional, numpy ndarray)
- ``metadata``: Metadata dictionary

Key Methods
^^^^^^^^^^^

- ``add_unit(uid)``: Add a memory unit to the space
- ``remove_unit(uid)``: Remove a memory unit from the space
- ``add_child_space(name)``: Add a subspace
- ``remove_child_space(name)``: Remove a subspace
- ``set_summary(text, embedding)``: Set space summary text and vector
- ``get_all_unit_uids(recursive=True, resolver=...)``: Recursively get all unit UIDs (requires resolver for recursion)
- ``get_all_child_space_names(recursive=True, resolver=...)``: Recursively get all subspace names
- ``to_dict()`` / ``from_dict(data)`` (classmethod): Serialization and deserialization
- ``touch()``: Update timestamp

Hierarchical Structure
^^^^^^^^^^^^^^^^^^^^^^

Spaces support a tree-like hierarchical structure, managed by ``SemanticMapService`` via ``SpaceNamingPolicy``:

.. code-block::

   root
   ├── root_base_memory
   │   └── [unit_001, unit_002, ...]
   └── root_high_level_memory
       ├── root_episodic
       │   ├── root_episodic_summary
       │   └── root_episodic_event
       ├── root_knowledge
       │   ├── root_knowledge_summary
       │   └── root_knowledge_entity
       ├── root_emotional
       ├── root_procedural
       └── root_insights

Usage Example
^^^^^^^^^^^^^

.. code-block:: python

   from mandol.domain.memory_space import MemorySpace
   from mandol.domain.types import SpaceName, Uid

   root = MemorySpace(name=SpaceName("root"))
   root.add_child_space(SpaceName("base_memory"))
   root.add_child_space(SpaceName("knowledge_entity"))
   root.add_unit(Uid("unit_001"))

.. _en-data-structure-semantic-map:

SemanticMap
-----------

``SemanticMapService`` (in ``mandol.application.semantic_map``) is the core service managing memory storage and vector indexing.
It integrates ``UnitStore`` (unit storage), ``AdaptiveVectorIndex`` (adaptive vector index), ``EmbeddingProvider`` (vectorization),
and ``Reranker`` (reranking), providing complete infrastructure for semantic retrieval.

.. note::

   **Naming**: The class in code is ``SemanticMapService``, accessed via ``system.semantic_map`` property.
   Documentation uses **SemanticMap** for brevity.

Constructor
^^^^^^^^^^^

.. code-block:: python

   SemanticMapService(
       *,
       store: UnitStore,
       index: VectorIndex,
       embedder: Optional[EmbeddingProvider] = None,
       reranker: Optional[Reranker] = None,
       default_text_key: str = "text_content",
       default_image_path_key: str = "image_path",
       max_units_in_memory: int = 10000,
   )

Core Interfaces
^^^^^^^^^^^^^^^

**Space Management**:

- ``create_space(name) -> MemorySpace``: Create a new space (returns existing if present)
- ``get_space(name) -> Optional[MemorySpace]``: Get a space by name
- ``list_spaces() -> List[MemorySpace]``: List all spaces
- ``add_unit_to_space(uid, space_name)``: Add a unit to a space
- ``attach_child_space(parent, child, *, ensure_exists=True)``: Establish parent-child space relationship
- ``ensure_child_space(parent, child) -> MemorySpace``: Ensure child space exists and attach

**Unit Management**:

- ``add_unit(unit, *, space_names, ensure_embedding=True, ...)``: Add a memory unit
- ``upsert_unit(unit, *, rebuild_index_immediately=False)``: Insert or update a unit
- ``delete_unit(uid)``: Delete a unit (removes from all spaces)
- ``get_unit(uid) -> Optional[MemoryUnit]``: Get a single unit
- ``list_units() -> List[MemoryUnit]``: List all units

**Space Retrieval**:

- ``get_units_in_spaces(space_names, *, mode="union", recursive=True) -> List[MemoryUnit]``: Get units within specified spaces

**Semantic Retrieval**:

- ``search_by_text(query_text, *, top_k=10, space_names=None, recursive=True) -> List[Tuple[MemoryUnit, float]]``: Text-based semantic search
- ``search_by_vector(query, *, top_k=10, space_names=None, recursive=True) -> List[Tuple[MemoryUnit, float]]``: Vector-based semantic search
- ``search_by_text_with_rerank(query_text, *, top_k=10, recall_k=None, space_names=None, recursive=True, use_rerank=True) -> List[Tuple[MemoryUnit, float]]``: Text-based search with reranking
- ``search_in_space(query_text, space_name, candidates=None, *, top_k=10, recall_k=None) -> List[Tuple[MemoryUnit, float]]``: Search within a specific space

**Index Maintenance**:

- ``rebuild_index_from_store()``: Rebuild vector index from storage
- ``set_embedder(embedder)``: Replace embedder
- ``set_reranker(reranker)``: Replace reranker
- ``flush()``: Clear all data

Usage Example
^^^^^^^^^^^^^

.. code-block:: python

   # Usually not manually created — managed by MemorySystem
   # Access via system.semantic_map

   custom_space = system.semantic_map.create_space("my_custom_space")
   system.semantic_map.add_unit_to_space(Uid("dialogue_001"), "my_custom_space")

   results = system.semantic_map.search_by_text("Beijing", top_k=10)
   entities = system.semantic_map.get_units_in_spaces(["root_knowledge_entity"])

SemanticGraph
-------------

``SemanticGraphService`` (in ``mandol.application.semantic_graph``) is the core service for managing graph relationship structures between memory units.
It consists of explicit relationships (entity relations, event causality, provenance, coreference, etc.) and implicit semantic relationships (vector similarity),
with graph storage delegated to ``GraphStore`` (default: ``InMemoryGraphStore``).

.. note::

   **Naming**: The class in code is ``SemanticGraphService``, accessed via ``system.graph`` property.
   Documentation uses **SemanticGraph** for brevity.

Constructor
^^^^^^^^^^^

.. code-block:: python

   SemanticGraphService(
       *,
       semantic_map: SemanticMapService,
       graph_store: GraphStore,
   )

Core Interfaces
^^^^^^^^^^^^^^^

**Unit Management**:

- ``add_unit(unit, *, space_names, ensure_embedding=True, ...)``: Add a unit (delegates to SemanticMap)
- ``delete_unit(uid)``: Delete a unit and all its edges

**Relationship Management**:

- ``add_relationship(source_uid, target_uid, relationship_name, **properties)``: Add a relationship edge
- ``get_relationship(source_uid, target_uid, relationship_name) -> Optional[Dict]``: Query relationship properties
- ``delete_relationship(source_uid, target_uid, relationship_name=None)``: Delete a relationship edge

**Graph Retrieval**:

- ``get_explicit_neighbors(uids, *, rel_type=None, direction="out") -> List[MemoryUnit]``: Get explicit neighbors
- ``get_implicit_neighbors(uids, *, top_k=10) -> List[Tuple[MemoryUnit, float]]``: Get implicit semantic neighbors
- ``get_units_in_spaces(space_names, *, mode="union", recursive=True) -> List[MemoryUnit]``: Get units in spaces
- ``bfs_expand_units(seeds, *, per_seed=3, hops=1, rel_type=None) -> List[MemoryUnit]``: BFS graph expansion

**Maintenance**:

- ``flush()``: Clear all graph data
- ``get_graph_store() -> GraphStore``: Get underlying graph store

Graph Structure Composition
^^^^^^^^^^^^^^^^^^^^^^^^^^^

SemanticGraph consists of two parts:

**Explicit Relationships**:

- Entity relation edges: ``RELATED_TO`` (with subtypes: hometown, lives_in, works_at, located_in, part_of)
- Event causal edges: ``CAUSES``, ``CAUSED_BY``
- Event-Entity edges: ``INVOLVES`` (with subtypes: participant, location, organizer, victim)
- Provenance edges: ``EVIDENCED_BY`` (pointing to original dialogue units)
- Coreference edges: ``COREF`` (cross-session merging of same-reference entities/events)
- Alias edges: ``ALIAS_OF`` (entity alias relationships)
