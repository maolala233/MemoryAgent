Core Concepts at a Glance
=========================

You don't need to memorize everything. Just understand these three concepts and you can start using Mandol.

MemoryUnit — The Smallest Unit of Memory
-----------------------------------------

Like a sticky note. Every conversation, every knowledge point, every event is a sticky note.

.. code-block:: python

   MemoryUnit(
       uid=Uid("msg_1"),                             # Unique identifier
       raw_data={
           "text_content": "Zhang San went to Beijing on a business trip",  # Text content (auto-vectorized)
           "image_path": "/path/to/photo.jpg",        # Image path (auto-vectorized, optional)
       },
       metadata={"timestamp": "2024-01-15T10:00"},   # Additional tags
   )

Key attributes:

- ``uid`` — Unique identifier
- ``raw_data`` — Raw content (``text_content`` and ``image_path`` are auto-vectorized, other fields are stored only)
- ``metadata`` — Additional tags (timestamps, sources, etc.)
- ``embedding`` — Dense vector (auto-generated by the system)
- ``sparse_embedding`` — Sparse vector (auto-generated by the system)

.. tip::

   ``text_content`` and ``image_path`` in ``raw_data`` are automatically vectorized by the system for retrieval. Other fields (like ``speaker``, ``source``) are stored as metadata but not auto-vectorized. ``metadata`` is for storing additional information like timestamps and sources for later filtering.

.. warning::

   Mandol currently only supports **plain text** and **image paths** as content formats. PDF, Markdown, Word and other files cannot be inserted directly — you need to extract them to plain text using external tools first, then put the text into ``text_content``. See :doc:`/shared/data-format-guide` for details.

For the complete attributes, methods, and interface signatures, see :doc:`/shared/data-structures-reference`.

MemorySpace — The Category Folder for Sticky Notes
---------------------------------------------------

Like a folder. You can put sticky notes in different folders, and folders can contain subfolders.

Key attributes:

- ``name`` — Space name
- ``unit_uids`` — Set of memory units contained
- ``child_spaces`` — Set of child spaces
- ``summary_text`` — Space summary text (optional)

.. code-block:: python

   # System-created space
   system.semantic_map.create_space("My Clients")

   # Add to space
   unit = MemoryUnit(
       uid=Uid("client_1"),
       raw_data={"text_content": "Li Si's company does cross-border e-commerce"},
   )
   system.add(unit, space_names=["My Clients"])

   # Later, you can search only within this space
   hits = system.retrieve_in_space("cross-border e-commerce", space_name="My Clients")

For the complete attributes, methods, and interface signatures, see :doc:`/shared/data-structures-reference`.

High-Level Memory — The System's Auto-Organization Capability
--------------------------------------------------------------

When you call ``build_high_level()``, the system automatically does four things:

1. **Identify topic boundaries**: Detects topic changes through LLM semantic analysis, grouping memories by semantic topic
2. **Find key elements**: Extracts people, places, concepts (entities) that appear
3. **Record events**: Marks important things that happened
4. **Write summaries**: Generates a summary for each semantic topic

All of this is automatic — you don't need to do anything extra.

.. code-block:: python

   system.build_high_level(mode="auto")
   # mode="auto": Only process new content (recommended for daily use)
   # mode="force": Clear and rebuild (use after tuning parameters)

.. note::

   "Session" does not equal "conversation." In Mandol, a Session is a semantic topic boundary — whether it's a conversation, document, or log, the system automatically groups by topic changes. Time intervals are only used as a reference for LLM judgment, not as a standalone trigger for segmentation.
