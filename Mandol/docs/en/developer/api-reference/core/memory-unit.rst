MemoryUnit Reference
========================

MemoryUnit is the smallest memory carrier in the system.

Constructor
------------

.. code-block:: python

   MemoryUnit(
       uid: Uid,
       raw_data: dict[str, Any],
       metadata: dict[str, Any] | None = None,
       unit_type: str | None = None,
   )

Fields
-------

.. list-table::
   :header-rows: 1
   :widths: 18 15 67

   * - Field
     - Type
     - Description
   * - ``uid``
     - Uid
     - Unique identifier
   * - ``raw_data``
     - dict
     - Raw content, flexible container for any key-value pairs; only ``text_content`` and ``image_path`` are auto-vectorized
   * - ``metadata``
     - dict
     - Additional tags, can store timestamp / speaker / source, etc.
   * - ``embedding``
     - Optional[numpy.ndarray]
     - Dense vector representation (generated by EmbeddingProvider)
   * - ``sparse_embedding``
     - Optional[numpy.ndarray]
     - Sparse vector representation (generated by SparseIndex)
   * - ``unit_type``
     - str
     - Auto-inferred (dialogue/knowledge/event, etc.)

Methods
--------

.. list-table::
   :header-rows: 1
   :widths: 25 15 60

   * - Method
     - Type

raw_data Details
-----------------

``raw_data`` has type ``Dict[str, Any]`` with no fixed schema. The system only auto-vectorizes the following fields:

Auto-Vectorized Fields
~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1
   :widths: 18 12 30 40

   * - Field Name
     - Type
     - Processing
     - Description
   * - ``text_content``
     - str
     - Dense Embedding + BM25 + Sparse index
     - Core field, almost all retrieval and build processes depend on it
   * - ``image_path``
     - str
     - ``embed_image_paths`` generates vectors
     - Interface reserved, current implementation falls back to path string encoding

Text-first principle: When both ``text_content`` and ``image_path`` are provided, only ``text_content`` is used.

Text Extraction Fallback Order
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

::

   text_content → text → content → summary → title → message → first string value

System-Internally Auto-Generated Fields
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1
   :widths: 22 30 48

   * - Field Name
     - Unit Type Where It Appears
     - Description
   * - ``text_content``
     - Entity, Event, Summary
     - Formatted text description
   * - ``entity_name``
     - Entity
     - Entity name
   * - ``entity_type``
     - Entity
     - Entity type (Person / Organization / Location, etc.)
   * - ``description``
     - Entity, Event
     - Detailed description
   * - ``summary``
     - Summary
     - Summary text
   * - ``type``
     - Summary
     - Summary type identifier
   * - ``insights``
     - Insights
     - List of insight contents
   * - ``entities``
     - Base unit
     - Field read by legacy entity extraction path

.. note::

   When extending ``raw_data``, avoid conflicting with system-internal field names. Always use ``text_content`` as the text field name, don't rely on the fallback mechanism.

For the complete field reference, see :doc:`/shared/data-format-guide`.

Usage Example
--------------

.. code-block:: python

   from mandol import MemoryUnit, Uid

   unit = MemoryUnit(
       uid=Uid("unique_id"),
       raw_data={
           "text_content": "A conversation or document content",
           "speaker": "Zhang San",
           "source": "WeChat",
       },
       metadata={"timestamp": "2024-01-15T10:00:00", "speaker": "Zhang San"},
   )
